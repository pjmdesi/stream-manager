## Improvement ideas

(Sorted easiest → hardest.)

1. When bulk-selecting then archiving a large number of stream items at once, it starts the conversion process for all of them at the same time which causes the app to freeze for a bit while they start.
2. The thumbnail editor overview page needs a consistency & loading update: The "Thumbnail Editor" header needs to be updated to match the other pages (use the streams page as reference). An obvious mismatch is the font size of the page title in the header. Additionally, while the template cards are loading, the page shows a simple "Loading..." text element. This needs to be updated to match loading states in the rest of the app, ie adding a spinner. Also for this templates section, when they load in, they cause the content below to jump since they render with large heights. We should reserve the space with min-height on the container. Also, when navigating from and back to the thumbnail editor overview page, the template cards seem to need to load in every time its visited. They should only need to load in once.
3. Expand the usage of the topic/game merge field to allow the user to select which game from the topics/tags field in the stream details is used for the merge field if they have multiple. This would allow the user to put multiple topics/games in the title & description fields. This would apply to the additional merge fields that are being added for the stream item details as well (hashtags, topic_details). The merge field chip would possibly have a dropdown next to its title when in the input field with the following options: "All" (renders all topics/games items in-line space-separated), "Primary" (default when the chip is added to a field; renders the *selected* topic/game item, best used for the Twitch title field), specific selection (The user would pick a specific topic/game item from the list of items in the topics/tags field for that stream item, best used for the YouTube title and description fields).
4. More merge fields for use in the stream item details:
    1. hashtags - these would link to the topic / game like the YouTube tags do. These would mostly be used in the YouTube description, but should be available for the title as well.
    2. topic_details - also linked to a topic / game tag, this would be a way to add details about a game or topic for things like links, description, etc. This would be used for the YouTube description, and would be a way to easily add (and update) the same snippet of text to multiple streams that are all about the same game or topic.
    3. streamer_links - this would be a way to add links to merch or other info about the streamer/channel. This would be used for the YouTube description, and would be a way to easily add (and update) the same snippet of text to multiple streams.
5. More keyboard shortcuts on the streams page. If a button does the same thing, add the shortcut to the tooltip for that button as smaller, slightly darker text as the last item in the tooltip. We should list the shortcuts in the help modal in their own category. We should move the current player shortcuts to that category as well under their own section. Global app shortcuts will have their own section and page-specific shortcuts will also go under respective sections. Some ideas for shortcuts:
    1. ctrl+up/down to navigate the stream items while the detail sidebar is open.
    2. ctrl+shift+r to start the reschedule process for a stream item while the detail sidebar is open.
    3. ctrl+shift+t to open the thumbnail editor for a stream item while the detail sidebar is open.
    4. ctrl+shift+up/down to navigate between SM pages (streams, player, converter, thumbnails) from anywhere in the app.
    5. ctrl+shift+w to start/stop the watcher anywhere in the app.
6. Make audio-track extraction more useful by letting the user choose which audio track to pull out, rather than being locked to the first track. The current "Extract Audio (First Track)" preset is limited because MP3 only supports a single track, so the picker / additional logic likely needs to live outside the Converter page (e.g. a per-track export option in the Player) — design is open. Possibly there could be a new button added to each audio track row in the player page when in the Multi-track mode, allowing the user to save the extracted audio as an MP3 file (or maybe others). These would be sent to the converter with any necessary parameters to specify which track to extract and in what format.
7. Maybe add the ability to connect and use other AI services in the integrations page besides Claude.
    1. Claude integration expansion: add an opt-in web search capability to the Claude AI suggestions. Claude supports the server-side `web_search` tool, which would let tag/title/description generation ground itself in real, current info (actual game titles, trending terminology, real channel categories) instead of guessing from training knowledge. This is bigger than a config toggle — the single request/response in `claude.ts` becomes a small agentic loop (handle `tool_use`/`pause_turn` stop reasons until done), web search is metered/billed per call, and it adds latency, so it should be gated behind a per-request "search the web" toggle rather than always-on. Model selection dropdown already shipped; this is the next lever for the "tag generation feels lacking" complaint.
8. Investigate if it's worth adding some performance enhancements to the stream page. Since the list of stream items can be long (I already have over 200) and each one has an image, we might need to lazy load the images. We could also load in stream items in batches instead of all at once when the user scrolls past a certain point. We would want the scrollbar to accurately reflect the full number of streams, but the actual content would load in dynamically as the user scrolls.
9. Upgrade the video counter tooltip to show more info such as the videos' thumbnails & encoding info. Add ability to delete video files from that tooltip. The new tooltip should be similar to the session videos section of the player page's sidebar. In fact we should create a new component for this type of tooltip that can be reused in both places. It will show a list of videos with their thumbnail, title, and encoding info, hydration status, timecode, allow the user to click on the filename to open it in the file explorer, and also have a delete button that sends the video to the recycle bin and removes it from the list.
10. Upgrade the carousel in the details sidebar to show all the media files in a stream folder, including videos and audio files (if any). When the user opens the full screen carousel view, they should be able to play the video/audio files with simple controls (play/pause, time scrubber, volume for audio). A button to "send to player" should appear for video files in the same style and placement as the "edit thumbnail" button. Video & audio files would not qualify for the "Set as Stream thumbnail" button, so that wouldn't appear for those. We should show 3 buttons in the top-left of the carousel element that allows to user to quickly show/hide certain types of files (one for images, one for video, one for audio) they are all "active" by default. When clicked, the button deactivates and the files are filtered out of the carousel view until the button is clicked again to reactivate it.
11. The converter page needs a layout update. The left sidebar conflicts with layout rules developed since this page was created. I think we need to update the presets sidebar to a presets management modal. This modal will match the layout & styling rules of other management modals in the app such as the templates management modal on the streams page. This also means we need to change how the user selects a preset. As an upgrade to the current method, we should move the "select preset" chip (currently present in the "ready files" panel at the top of converter page main content) to the individual items in that ready files list. Each item will have its own "select preset" dropdown instead. Additionally, we need to add the same "default" system also present in the templates modal which will indicate which preset is the default for new conversion items added to the page. This should use the star design pattern. This means the "recommended" start needs to update to use another icon. So there will be 3 icons used for the presets: the "default" star, the "recommended" badge (maybe a thumbs up or something), and the "default for archiving" Archive icon. Since the presets will exist in a modal, the "new custom preset" button functionality will be updated to be inside that modal. In the same way that adding a new template is done in the templates management modal (the form appears directly on the page, the modal might need to be extra tall to accommodate the size of the form, we could also pair up some of the input fields into 2 columns to save some vertical space, use your best judgement).
12. The launcher page needs an update. The left sidebar doesn't match the layout rules developed since this page was created. Instead the groups should be what appears (as rows similar to stream items on the streams page) in the main content and opening a group should open a right sidebar similar to how the stream item details sidebar works. This will then show what the main content of the launcher page currently shows. The main content of the page also needs a header matching the layout/style of the other pages in the app (use the streams page for reference). The app group list items should include as columns: The icon for the group, the name of the group, the icons for the apps in the group, a button to launch the whole group, the star icon indicating that this is the default group for the launcher widget in the navigation sidebar. Clicking one of the icons from the row should launch that individual app. When the sidebar opens, it would expand to obscure all but the icon and name of the group item rows (similar to how the stream detail sidebar obscures all but the thumbnail, video counter, and title columns when it expands). Right now there's no "closed" version of the sidebar for the launcher page, so suggestions are welcome. If nothing makes sense, the sidebar will just not appear unless a group row is clicked.
13. Add the ability for the launcher to track which of the apps in each launch group are still open and allow the user to quit them from the launcher. Need to discuss design. We could also add more options to the launcher group items after this such as 2 boolean options: Close with group (checked by default, unchecked means it won't quit when the user clicks the "Quit Group", for example, an app that the user would like to keep open after streaming), and Allow multiple instances (unchecked by default, checked means the app could be attempted to be launched multiple times when the group or individual launch buttons are clicked. Might need to check if it's possible to know if an app can have multiple instances so there's a smaller chance of conflict. If we can, the checkbox would not appear for those apps).
14. Bulk editing of stream items. For instance, let's say the streamer has finished a game and they want to now add the "{total_episodes}" merge tag to all the stream items in that season. They could select all the items for that season and then have the option to bulk edit the title template for all those items at once. SM would swap the templates for all those streams, update the titles, and then offer to push the changes to YouTube for all those streams at once as well.
15. Add the ability for SM to create a new stream item based on an already existing YouTube LIvestream, broadcast, or regular video. It will read and fill in the details for the stream item based on the YouTube video's details. This would greatly help out new users who already have a lot of videos. It would be good to be able to batch this process as well. Might run into API quotas. Need to discuss.
16. Add the ability for users to create stream items by pulling videos/VODs directly from YouTube. This would be a great way for new users to quickly populate their stream list with existing content. The user would need to connect their YouTube account and then they could select from a list of their existing videos or livestreams to create stream items from or download all at once (for initial population). This would pull in the details for each video and fill in the stream item details based on that, including the thumbnail. This would probably be a modal found in the YouTube section of the integrations page. SM would create new folders in the root dir for each stream based on the upload/stream date & add any necessary files (like the thumbnail) to those folders.
17. Add shorts upload functionality. Needs to be able to upload to YouTube.
18. Add "clip on twitch" functionality. This is a bit more complex than the YouTube one, because Twitch clips are created through an API call that takes a start and end time, and then Twitch processes the clip and makes it available after a few minutes. We would need to have some kind of system for checking the status of the clip creation and updating the app when it's ready. We also need to make sure the user knows the limitations of twitch clips.
19. Add the ability for the app to update on its own without having to send the user to the GitHub releases page to download the new version. This would be a big improvement for user experience. Maybe there's a library that can handle this. We would need to set up a system for hosting the updates, and then the app would need to check for updates on startup and prompt the user to download and install (or skip) the new version if one is available (would show the release notes for the new version and any versions in between the new one and their current one).
20. Create a Stream Dashboard pop-out window. This would essentially be a trimmed-down controller for SM which would show info essential for the active stream. This would be one way to show the chat messages in the item above. Additionally it could show the relay status and stats. Also the user would be able to use this to change the stream details live for all platforms at the same time (for instance if they switch games mid-stream, or come up with a better title while streaming). Not sure if this should be a separate exe that the user ca launch separately from SM if they want (it would read the same _meta.json for the info it needs) or if SM should have to be open to have it work. Would need to think through that. Would need a minimal, but effective layout & design.
21. Build a chat viewer that can connect to the YouTube and Twitch APIs to show the live chat for the active broadcast. This would be a new page in the app that shows the chat messages in real time, along with some basic info about the chat such as the number of viewers for each platform, and maybe some basic moderation tools like the ability to delete messages or ban users.

## Other ideas (big)

1. Maybe a music folder manager as well? To control and add info to the music that's played during streams through OBS.
   * Not sure if it should play through the SM app or just manage the files and the metadata for the music that's played through OBS.
   * Maybe it could serve a custom web page that displays in OBS as a browser source that shows the current track info and maybe even the album art or something in a customizable layout.
   * The biggest add for me would be a way to group music items into playlists so I can easily switch between different sets of music for different types of streams or different moods. This way, I won't have to manually edit the settings of the OBS music source every time I want to change up the music.
   * Maybe one day, there could even be a feature that allows the user's audience to vote on what music they want to hear during the stream through a Twitch extension or a website. And maybe even rate the music or suggest new tracks to add to the playlists. This would be a fun way to engage with the audience and make the music selection more interactive.
2. Add the ability to switch between different YouTube channels from within SM. This would mean switching the root folder that SM is monitoring for stream items, as well as switching the YouTube account that SM is connected to for uploading and managing the streams. This would be useful for users who manage multiple channels or want to separate their content into different channels. The user would need to go through the YouTube integration process for each channel they want to connect, and then they could easily switch between them in the app through a dropdown in the app window header next to the app title. Not sure how this would affect twitch, maybe the option to tie a particular Twitch account to each YouTube channel so that when you switch channels, it also switches the Twitch account that's connected for the stream relay and chat features.
3. For the thumbnail editor: add the ability to search for and insert graphics related to a game. Utilizing a service such as SteamDB & SteamGridDB or another game database API, the user could search for a game and then pull in official assets like logos, character renders, screenshots or other promotional images to use in their thumbnails. This would save time for streamers who don't want to have to search directly.
