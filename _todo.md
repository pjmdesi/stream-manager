## Improvement ideas

(Sorted easiest → hardest.)

1. More keyboard shortcuts on the streams page. If a button does the same thing, add the shortcut to the tooltip for that button as smaller, slightly darker text as the last item in the tooltip. We should list the shortcuts in the help modal in their own category. We should move the current player shortcuts to that category as well under their own section. Global app shortcuts will have their own section and page-specific shortcuts will also go under respective sections. Some ideas for shortcuts:
    1. ctrl+up/down to navigate the stream items while the detail sidebar is open.
    2. ctrl+shift+r to start the reschedule process for a stream item while the detail sidebar is open.
    3. ctrl+shift+t to open the thumbnail editor for a stream item while the detail sidebar is open.
    4. ctrl+shift+up/down to navigate between SM pages (streams, player, converter, thumbnails) from anywhere in the app.
    5. ctrl+shift+w to start/stop the watcher anywhere in the app.
2. Now that the mismatch functionality is more built out, maybe that's what can display beneath the calendar in the stream page sidebar, a list of all the stream items with a mismatch (and maybe what the mismatch is). We could even have buttons that bulk resolve the mismatches for all the streams in the list or for individual stream items in the list. Clicking an item in the list would open that item in the detail sidebar for the user to investigate if they so desire. The bulk buttons would be dangerous, so they should have a confirmation step that lists what changes would be made and requires the user to type "confirm" to proceed.
3. Upgrade the video counter tooltip to show more info such as the videos' thumbnails & encoding info. Add ability to delete video files from that tooltip. The new tooltip should be similar to the session videos section of the player page's sidebar. In fact we should create a new component for this type of tooltip that can be reused in both places. It will show a list of videos with their thumbnail, title, and encoding info, hydration status, timecode, allow the user to click on the filename to open it in the file explorer, and also have a delete button that sends the video to the recycle bin and removes it from the list.
4. Upgrade the carousel in the details sidebar to show all the media files in a stream folder, including videos and audio files (if any). When the user opens the full screen carousel view, they should be able to play the video/audio files with simple controls (play/pause, time scrubber, volume for audio). A button to "send to player" should appear for video files in the same style and placement as the "edit thumbnail" button. Video & audio files would not qualify for the "Set as Stream thumbnail" button, so that wouldn't appear for those. We should show 3 buttons in the top-left of the carousel element that allows to user to quickly show/hide certain types of files (one for images, one for video, one for audio) they are all "active" by default. When clicked, the button deactivates and the files are filtered out of the carousel view until the button is clicked again to reactivate it.
5.  Investigate if it's worth adding some performance enhancements to the stream page. Since the list of stream items can be long (I already have over 200) and each one has an image, we might need to lazy load the images. We could also load in stream items in batches instead of all at once when the user scrolls past a certain point. We would want the scrollbar to accurately reflect the full number of streams, but the actual content would load in dynamically as the user scrolls.
6.  Add the ability for the launcher to track which of the apps in each launch group are still open and allow the user to quit them from the launcher. Need to discuss design. We could also add more options to the launcher group items after this such as 2 boolean options: Close with group (checked by default, unchecked means it won't quit when the user clicks the "Quit Group", for example, an app that the user would like to keep open after streaming), and Allow multiple instances (unchecked by default, checked means the app could be attempted to be launched multiple times when the group or individual launch buttons are clicked. Might need to check if it's possible to know if an app can have multiple instances so there's a smaller chance of conflict. If we can, the checkbox would not appear for those apps).
7.  Make audio-track extraction more useful by letting the user choose which audio track to pull out, rather than being locked to the first track. The current "Extract Audio (First Track)" preset is limited because MP3 only supports a single track, so the picker / additional logic likely needs to live outside the Converter page (e.g. a per-track export option in the Player) — design is open. Possibly there could be a new button added to each audio track row in the player page when in the Multi-track mode, allowing the user to save the extracted audio as an MP3 file (or maybe others). These would be sent to the converter with any necessary parameters to specify which track to extract and in what format.
8.  Update the youtube description field in the details sidebar to use the same merge field chips system that the YouTube title. This will need to include a mode switcher that switches between the chips template mode and a plain text mode for the description since the description can be quite long and have a lot of merge fields, which might make the chips view a bit overwhelming.
9.  Bulk editing of stream items. For instance, let's say the streamer has finished a game and they want to now add the "{total_episodes}" merge tag to all the stream items in that season. They could select all the items for that season and then have the option to bulk edit the title template for all those items at once. SM would swap the templates for all those streams, update the titles, and then offer to push the changes to YouTube for all those streams at once as well.
10. Add the ability for SM to create a new stream item based on an already existing YouTube LIvestream, broadcast, or regular video. It will read and fill in the details for the stream item based on the YouTube video's details. This would greatly help out new users who already have a lot of videos. It would be good to be able to batch this process as well. Might run into API quotas. Need to discuss.
11. Add shorts upload functionality. Needs to be able to upload to YouTube.
12. Add "clip on twitch" functionality. This is a bit more complex than the YouTube one, because Twitch clips are created through an API call that takes a start and end time, and then Twitch processes the clip and makes it available after a few minutes. We would need to have some kind of system for checking the status of the clip creation and updating the app when it's ready. We also need to make sure the user knows the limitations of twitch clips.
13. Build a chat viewer that can connect to the YouTube and Twitch APIs to show the live chat for the active broadcast. This would be a new page in the app that shows the chat messages in real time, along with some basic info about the chat such as the number of viewers for each platform, and maybe some basic moderation tools like the ability to delete messages or ban users.
14. Create a Stream Dashboard pop-out window. This would essentially be a trimmed-down controller for SM which would show info essential for the active stream. This would be one way to show the chat messages in the item above. Additionally it could show the relay status and stats. Also the user would be able to use this to change the stream details live for all platforms at the same time (for instance if they switch games mid-stream, or come up with a better title while streaming). Not sure if this should be a separate exe that the user ca launch separately from SM if they want (it would read the same _meta.json for the info it needs) or if SM should have to be open to have it work. Would need to think through that. Would need a minimal, but effective layout & design.
15. When bulk-selecting then archiving a large number of stream items at once, it starts the conversion process for all of them at the same time which causes the app to freeze for a bit while they start.
16. We might need to mirror the YouTube tag auto-sorting functionality in the app as well. Currently, if the user adds tags to the stream item in SM, they will be added in whatever order the user typed them in, but when they are pushed to YouTube, YouTube will automatically sort them alphabetically. This would make it clear that the tags are the same in both places and avoid confusion for the user when they see the tags in SM in a different order than they see them on YouTube. The sort should happen when the field is blurred.
